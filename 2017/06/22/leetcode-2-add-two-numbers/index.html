[LeetCode] 2. Add Two Numbers

這次來做的是Algorithms的第二題，難度為Medium，

題目要求是傳入兩個代表某非負整數的linked list(node value為one digit的非負整數)，

回傳相加後的結果，型態和傳入的linked list一樣。

範例(取自LeetCode)：

假設執行342+465要計算出807這個值，

則

Input: (2 -> 4 -> 3) + (5 -> 6 -> 4)
Output: 7 -> 0 -> 8

(題目連結)

首先一樣依照TDD的模式，

先寫好測試案例，

附帶一提，題目中也有定義好要使用的linked list類別

https://gist.github.com/yohey03518/3f0754948de66e454699b6919873855d

測試初始化

https://gist.github.com/yohey03518/9ccd4e47c615bf7d592bbc6a11299f53

第一個測試案例：單位數的相加，並且相加結果不超過10

https://gist.github.com/yohey03518/7759a430be66d09ef51e0f50c617343a

 

接著實作method AddTwoNumbers滿足此需求

https://gist.github.com/yohey03518/3661ea1b2b14c800677b44861cd7d1a7

 

很好，測試通過了

接著下一個測試案例：兩個單位數相加大小會超過十(雙位數)

https://gist.github.com/yohey03518/289627c3cdb4e854f49035d21a70c4e6

想當然執行測試會失敗，趕緊來撰寫符合此兩規則的程式：

判斷若結果超過十，則要分位數去處理並回傳含有兩個節點的結果

https://gist.github.com/yohey03518/49bc635dd57a5c58e2af3da51e276a8e

的確測試也通過了

 

 

接著第三個測試案例，不同位數的相加但不涉及進位，

被加數和加數顛倒來當作不同測試案例

https://gist.github.com/yohey03518/734bd7846ff3913f7d4187a7908d5a89

這邊就要開始思考該用何種方式去爬所有的位數並做運算，

想法是在做加法時都會個位數十位數對齊去進行，

那就分別各用一個指標去走訪兩個input值，

去做個位數相加、十位數相加…以此類推，

這想法跟原先的寫法有些不同，因此要來重構一下

https://gist.github.com/yohey03518/473718e59c141a07bca4e355aebe0677

比較特別的是為了讓儲存加總結果的list可以順利讓指標走訪，

必須先new一個起始點，因此最終的結果是從resultStartPoint的next開始回傳。

 

好的那馬上來執行測試…

Oops！最新的測試案例是通過了，

但在我們的有涉及進位的第二項測試案例上卻是出錯的，

不過也沒什麼好難過的，TDD的過程就是滿足當下的需求以及前項的測試案例，

因此只要再調整一下讓它滿足其餘測試案例即可，

意即要針對前項實作的Line23針對可能的進位做處理，

程式中加上另一個變數來處理進位，

實作修改如下

https://gist.github.com/yohey03518/df43945d34a69972c7795e05c44b6195

測試也順利都通過了

接著下一個測試案例，多重進位，如3+99這種狀況會有兩次的進位

https://gist.github.com/yohey03518/f0ae3896c52e479694f29393129de840

 

結果發現，運氣不錯！先前的案例似乎也包含著這樣的狀況，測試全通過

嗯！總感覺是時候了，來Submit這一段code試試看吧！

結果出乎意料地順利呢！

打敗了88.67%的玩家，看來這數字某種程度上真的是個成就感呢！

那.. 今晚就來睡個好覺啦！

 

後記：

其實這一題在做的時候已經很晚接近不得不睡的時間了(還是個上班族RRR)，

於是就是不斷的trial and error，修改一次後就submit一次，

來來回回不知道總共幾遍，讓心情越來越毛躁，

看來也是個意外的收穫及經驗呀！
