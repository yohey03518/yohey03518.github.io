[LeetCode] 1. Two Sum

最近終於把家裡這台PC給處理好了，

那在這個失去熱情失去人生方向的小小低潮中，

來做做LeetCode嘗試點不一樣的事吧！

“Two Sum”這題是Algorithms項目中難度為Easy且為序號1的題目。

目標是傳入一個int陣列以及目標數字(int)，

使得陣列中某兩元素值相加可等於目標數字，

並且回傳此兩值在此陣列中的索引。

範例：(取自LeetCode)

Given nums = [2, 7, 11, 15], target = 9,

Because nums[0] + nums[1] = 2 + 7 = 9,
return [0, 1].
而最近粗淺的學習了TDD的開發方式，

也參考了91大以TDD的模式來解LeetCode題目的方式，

([TDD]LeetCode 58. Length Of Last Word)

那就來試著TDD吧！

接著為了滿足測試的最小需求，簡單的撰寫code：

public int[] GetSolution(int[] nums, int target)
{
if(nums[0] + nums[1] == target)
return new int[]{ 0, 1 };
else
throw new Exception(“No match result!”);
}
[/code]

在這裡提倡一下Visual Studio的指令Ctrl+R, A，真的是個好用的指令，

多重的功用已經成為了小弟我寫完程式要按建置的替代方式了！

(其實關於TDD的節奏真的拿捏的還不是很到位呀～不過就慢慢練習囉)

相當然爾執行測試會是失敗的，所以要來重構原先的代碼，

想法：恩… 要達成這個目標，一定要讓所有數字兩兩相加比對後才知道結果，

那就用兩個for迴圈把它解決掉吧！

[code language=”csharp”]
public int[] GetSolution(int[] nums, int target)
{
for (int index1 = 0; index1 < nums.Count(); index1++)
{
for (int index2 = (index1 + 1); index2 < nums.Count(); index2++)
{
if (nums[index1] + nums[index2] == target)
return new int[] { index1, index2 };
}
}
throw new Exception(“No match result!”);
}
[/code]

於是神奇的事情就這麼發生了，各位觀眾，四個測試案例都通過！

那還不趕快來Submit這段Code然後好好的去睡個覺嗎？

結果… 燈冷～紅燈啦！！

原來在LeetCode中，不是只有單純的成功跟失敗而已，

包含時間複雜度的因素也必須考量在內(難怪寫出兩個for迴圈時心裡總覺得毛毛der)，

當測試資料筆數變多時(陣列大小到達一萬多筆)，會超過執行時間的限制。

那當然就要再來重構一下解法啦，希望可以將原先複雜度Ｏ(n平方)改為Ｏ(n)，

 

想了一陣子，目標就是只可以走訪陣列一次，那就必須有另一個地方紀錄被走訪的值，

這邊使用C#的Map型態–Dictionary來進行，

根據走訪過的值，Map的Key值存入此被走訪到的值還需要加多少才可以達成target，

Value則存放此走訪的值的索引，

(也就是假設陣列索引走訪到3，值為5，目標為7，

則Map存放<7-5(期望之後要有的值), 3(索引值)>)

 

[code language=”csharp”]
public int[] GetSolution(int[] nums, int target)
{
// First int means the excepted second value, the second int means the index of first value
Dictionary exceptValueMap = new Dictionary();
for (int index1 = 0; index1 < nums.Count(); index1++)
{
// the value in array matches the excepted value
if (exceptValueMap.ContainsKey(nums[index1]))
{
return new int[] { exceptValueMap[nums[index1]], index1 };
}
else
{
exceptValueMap.Add(target - nums[index1], index1);
}
}
throw new Exception("No match result!");
}
[/code]

 

好的，測試也順利地通過了，趕快來Submit一下：

結果…

在某項測試案例上發生了字典Key值重複的問題，

趕緊檢視一下測試資料，發現陣列中真的有出現重複的值(這應該是題目中沒提到的)，

想了一下其實這種狀況在軟體專案的開發中還蠻常見的，

除了需求的變動外，例外狀況也常會是開發階段考慮不到的。

但透過單元測試的保護，即使出錯了，只要將測試案例補上並適度修改程式碼，

那就可確保未來不會再遇到相同的錯誤了。

 

因此簡單的加入判斷式以調整程式碼：

https://gist.github.com/yohey03518/bfcb58fcc9b6328073c070d2c303be2e


修改完成後，執行測試(包含最後因Key值重複的測試案例)，




皆顯示成功，接著Submit：




成功啦！而且居然有如此詳細的資訊，還可以跟使用者說明執行時間打敗了多少人，


嗯～打敗了85.46%的玩家，看來今晚可以睡個好覺了呢！


 


後記：


其實在LeetCode上Key值會發生重複的那一項測試案例中，


並沒有顯示預期Return的結果，


反而小弟我是使用先前兩個for迴圈的那段舊程式碼來找出結果後，


才組成測試案例供重構後的程式碼驗證用，
